# Design: unify-news-center

## Layout Changes

### 1. 资讯中心（/news）

```
┌─────────────────────────────────────────────────────┐
│  资讯中心                                            │
│  ┌──────────┬──────────┬──────────┐                 │
│  │ 新闻快讯  │ 机构调研  │ 研报数据  │               │
│  └──────────┴──────────┴──────────┘                 │
│                                                     │
│  新闻快讯 Tab:                                       │
│  ┌────────────────────────────┬──────────────┐      │
│  │ 工具栏: [来源▼][时间▼][分类▼] │  热点话题    │      │
│  │ ───────────────────────── │  情绪分析    │      │
│  │ 新闻卡片列表 + 滚动加载     │              │      │
│  └────────────────────────────┴──────────────┘      │
│                                                     │
│  机构调研 Tab: 复用 InstitutionalSurveyPanel         │
│  研报数据 Tab: 复用 ResearchReportPanel              │
└─────────────────────────────────────────────────────┘
```

### 2. 财报分析（/research）

```
┌──────────────────────────────┐
│  财报分析                      │
│  ┌──────────┬──────────┐      │
│  │ A股财报   │ 港股财报   │      │
│  └──────────┴──────────┘      │
└──────────────────────────────┘
```

## Key Decisions

1. **不改后端接口**：机构调研与研报继续通过插件路由提供数据。
2. **Tab 懒加载**：机构调研与研报 Tab 使用 `defineAsyncComponent` 延迟加载。
3. **筛选内联化**：新闻筛选项（来源/时间/分类）移动到新闻列表顶部工具栏，减少页面固定占位。
4. **入口保留**：`/research` 保留但更名为"财报分析"，避免用户路径断裂。

## Non-Goals

- 不调整新闻数据源或后台逻辑
- 不新增跨数据源的统一搜索
- 不新增新的筛选维度
