## ADDED Requirements

### Requirement: 研究数据综合页面

系统 SHALL 提供 `/research` 页面，整合机构调研、研报数据和财报研读功能。

#### Scenario: 查看机构调研数据
- **WHEN** 用户访问研究数据页面并选择"机构调研" Tab
- **THEN** 系统显示机构调研数据表格
- **AND** 表格包含：股票代码、名称、调研日期、调研机构、接待方式、调研内容等字段
- **AND** 用户可通过日期选择器筛选特定日期的数据
- **AND** 显示热门调研股票排名

#### Scenario: 查看研报数据
- **WHEN** 用户访问研究数据页面并选择"研报数据" Tab
- **THEN** 系统显示研报数据表格
- **AND** 表格包含：股票代码、名称、报告日期、标题、研究机构、分析师、评级、EPS预测、目标价等字段
- **AND** 用户可通过日期选择器筛选特定日期的数据
- **AND** 显示热门研报覆盖股票排名

#### Scenario: 查看财报研读
- **WHEN** 用户访问研究数据页面并选择"财报研读" Tab
- **THEN** 系统显示财报分析功能
- **AND** 功能与原独立财报研读页面保持一致
- **AND** 包含股票搜索、财务表格、趋势图表、AI分析等功能

### Requirement: 行情分析页面龙虎榜整合

系统 SHALL 在行情分析页面 `/market` 中提供龙虎榜分析功能。

#### Scenario: 通过 Tab 切换访问龙虎榜
- **WHEN** 用户在行情分析页面点击"龙虎榜" Tab
- **THEN** 系统显示龙虎榜数据和分析功能
- **AND** 功能与原独立龙虎榜页面保持一致
- **AND** 包含龙虎榜数据、投资组合分析、异动预警、市场统计

#### Scenario: 默认展示市场概览
- **WHEN** 用户首次访问行情分析页面
- **THEN** 系统默认显示"市场概览" Tab 内容
- **AND** 保持现有市场概览功能不变

### Requirement: 导航菜单更新

系统 SHALL 更新导航菜单以反映页面结构变化。

#### Scenario: 导航到研究数据页面
- **WHEN** 用户点击导航菜单中的"研究数据"
- **THEN** 系统跳转到 `/research` 页面

#### Scenario: 移除独立入口
- **WHEN** 用户查看导航菜单
- **THEN** 不显示独立的"龙虎榜分析"和"财报研读"菜单项
- **AND** 龙虎榜功能可通过"行情分析"页面的 Tab 访问
- **AND** 财报研读功能可通过"研究数据"页面的 Tab 访问
